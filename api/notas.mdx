Notes (`notas`) are timestamped annotations attached to a work order. They capture free-text observations, the state of the order at the time of writing, and optional images taken in the field. Every time a technician records progress or an issue, a nota is the right place to store it.

**Base path:** `/api/notas`

<Note>
  All note endpoints require a valid `Authorization: Bearer <token>` header.
</Note>

---

## POST /api/notas

Create a new note on a work order. Notes are linked to both the pendiente and the employee who wrote them, and they capture the current state of the order for historical context.

### Request body

<ParamField body="pendienteId" type="number" required>
  ID of the work order this note belongs to.
</ParamField>

<ParamField body="empleadoId" type="number" required>
  ID of the employee authoring the note.
</ParamField>

<ParamField body="nota" type="string" required>
  The note text. Supports free-form content — describe observations, actions taken, or next steps.
</ParamField>

<ParamField body="estadoPendiente" type="string" required>
  Snapshot of the work order's state at the time of writing. One of: `REGISTRADO`, `ASIGNADO`, `EN_PROGRESO`, `PAUSADO`, `REVISION`, `POR_VALIDAR`, `OBSERVADO`, `FINALIZADO`, `POSTERGADO`, `CANCELADO`.
</ParamField>

### Response — 200 OK

Returns the created `PendienteNotaResponse`:

<ResponseField name="id" type="number">Note ID.</ResponseField>
<ResponseField name="pendienteId" type="number">Parent work order ID.</ResponseField>
<ResponseField name="empleadoId" type="number">Author employee ID.</ResponseField>
<ResponseField name="nota" type="string">Note text.</ResponseField>
<ResponseField name="estadoPendiente" type="string">Work order state captured at creation time.</ResponseField>
<ResponseField name="fechaCreacion" type="string">ISO 8601 timestamp of creation.</ResponseField>
<ResponseField name="imagenes" type="PendienteNotaImagenResponse[]">
  Array of images attached to this note (empty on creation, populated via the upload endpoint).
  <Expandable title="PendienteNotaImagenResponse fields">
    <ResponseField name="id" type="number">Image ID.</ResponseField>
    <ResponseField name="notaId" type="number">Parent note ID.</ResponseField>
    <ResponseField name="url" type="string">URL to the stored image file.</ResponseField>
  </Expandable>
</ResponseField>

### Example

```bash cURL
curl --request POST \
  --url https://api.example.com/api/notas \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "pendienteId": 42,
    "empleadoId": 9,
    "nota": "Llegué al domicilio. El splitter externo está dañado. Procedo con el reemplazo.",
    "estadoPendiente": "EN_PROGRESO"
  }'
```

## GET /api/notas/{id}

Retrieve a single note by ID, including its attached images.

<ParamField path="id" type="number" required>Note ID.</ParamField>

```bash cURL
curl --request GET \
  --url https://api.example.com/api/notas/15 \
  --header 'Authorization: Bearer <token>'
```

## PUT /api/notas/{id}

Update an existing note's text or state snapshot.

<ParamField path="id" type="number" required>ID of the note to update.</ParamField>

Send the same fields as `POST /api/notas` with the values you want to update.

```bash cURL
curl --request PUT \
  --url https://api.example.com/api/notas/15 \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "pendienteId": 42,
    "empleadoId": 9,
    "nota": "Splitter reemplazado. Señal estable en -22 dBm. Entregado al cliente.",
    "estadoPendiente": "POR_VALIDAR"
  }'
```

## DELETE /api/notas/{id}

Soft-delete a note. The record is retained in the database but excluded from normal queries.

<ParamField path="id" type="number" required>ID of the note to delete.</ParamField>

```bash cURL
curl --request DELETE \
  --url https://api.example.com/api/notas/15 \
  --header 'Authorization: Bearer <token>'
```

## GET /api/notas/pendiente/{pendienteId}

List all notes for a work order, ordered by `fechaCreacion` ascending. Each note includes its attached images.

<ParamField path="pendienteId" type="number" required>
  ID of the work order whose notes you want to retrieve.
</ParamField>

```bash cURL

  --url https://api.example.com/api/notas/pendiente/42 \
  --header 'Authorization: Bearer <token>'
```

## POST /api/notas/{notaId}/imagenes/upload

Upload one or more images to an existing note. This endpoint uses `multipart/form-data` encoding — not JSON. Use this to attach photos taken in the field (e.g. before/after shots of the installation).

<ParamField path="notaId" type="number" required>
  ID of the note to attach images to.
</ParamField>

<ParamField body="files" type="file[]" required>
  One or more image files. Use the field name `files` for each file part. Supported formats depend on server configuration (typically JPEG, PNG).
</ParamField>

### Response — 200 OK

Returns an array of `PendienteNotaImagenResponse` objects for the newly uploaded images.

<ResponseField name="id" type="number">Image ID.</ResponseField>
<ResponseField name="notaId" type="number">Parent note ID.</ResponseField>
<ResponseField name="url" type="string">Public URL to access the stored image.</ResponseField>

<Warning>
  Do not send a `Content-Type: application/json` header with this request. The browser or HTTP client must set the boundary for multipart automatically.
</Warning>

### Examples

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.example.com/api/notas/15/imagenes/upload \
  --header 'Authorization: Bearer <token>' \
  --form 'files=@/path/to/photo1.jpg' \
  --form 'files=@/path/to/photo2.jpg'
```

```javascript JavaScript (fetch)
const formData = new FormData();
formData.append('files', file1);   // File object from <input type="file">
formData.append('files', file2);

const response = await fetch('https://api.example.com/api/notas/15/imagenes/upload', {
  method: 'POST',
  headers: {
    // Do NOT set Content-Type here — fetch sets it automatically with the boundary
    Authorization: `Bearer ${token}`,
  },
  body: formData,
});

const imagenes = await response.json();
```

</CodeGroup>

## GET /api/notas/{notaId}/imagenes

Retrieve all images attached to a specific note.

<ParamField path="notaId" type="number" required>Note ID.</ParamField>

### Response — 200 OK

Returns an array of `PendienteNotaImagenResponse` objects.

```bash cURL
curl --request GET \
  --url https://api.example.com/api/notas/15/imagenes \
  --header 'Authorization: Bearer <token>'
```

---

## PUT /api/notas/{notaId}/imagenes

Update the image records associated with a note. This endpoint accepts a JSON array of image request objects, not multipart form data.

<ParamField path="notaId" type="number" required>Note ID.</ParamField>

Send a JSON array of `PendienteNotaImagenRequest` objects in the request body.

```bash cURL
curl --request PUT \
  --url https://api.example.com/api/notas/15/imagenes \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '[{"url": "/uploads/notas/photo1.jpg"}, {"url": "/uploads/notas/photo2.jpg"}]'
```
